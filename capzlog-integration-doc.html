01 — Übersicht

NTA SkyNav ist eine Echtzeit-Navigationsapp für Ballonpiloten, verfügbar für iOS und Android. Die App zeichnet vollständige Flugdaten auf — GPS-Tracks, barometrische Daten, Gasverbrauch, Beladungsrechnungen und alle Informationen, die für das offizielle Fahrtenbuch benötigt werden.

02 — Flugdatenmodell

Jeder Flug wird als vollständiger Snapshot in der Supabase-Datenbank gespeichert. Nachfolgend alle erfassten Felder, kategorisiert nach ihrer Relevanz für das offizielle Fahrtenbuch.

Fahrtenbuch-Pflichtfelder

FeldTypBeschreibung
Datum & UhrzeitISO 8601Start- und Ende-Zeitstempel des Flugs
DauerZahl (ms)Berechnet aus Start → Ende
StartplatzGPS + NameKoordinaten + Reverse-Geocoding Ortsname
LandeplatzGPS + NameKoordinaten + Reverse-Geocoding Ortsname
PilotStringPilotenname (aus Benutzereinstellungen)
BallonObjektHersteller, Hüllentyp, Volumen (m³)
PassagiereZahlAnzahl der Passagiere an Bord
Max. HöheZahl (m MSL)Maximale erreichte Höhe während des Flugs
GasverbrauchArrayPro Flasche: Name, Kapazität (L), verbraucht (L), Rest (L)

03 — Datenbankschema

Flugdaten sind in zwei Supabase-Tabellen mit Row-Level-Security organisiert. Jeder Benutzer hat nur Zugriff auf seine eigenen Daten.

Supabase-Tabellen

-- Fluggruppen (Meisterschaften oder Einzelflüge)
CREATE TABLE championships (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id     UUID REFERENCES auth.users(id),
  name        TEXT NOT NULL,
  map_id      TEXT,
  archived    BOOLEAN DEFAULT false,
  created_at  TIMESTAMPTZ DEFAULT now()
);

-- Einzelne Flüge mit vollständigem Daten-Snapshot
CREATE TABLE championship_flights (
  id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  championship_id   UUID REFERENCES championships(id),
  name              TEXT NOT NULL,
  flight_data       JSONB,        -- vollständiger Flug-Snapshot
  created_at        TIMESTAMPTZ DEFAULT now()
);

04 — Integrationsoptionen

Unser Vorschlag für eine nahtlose Integration zwischen NTA SkyNav und CapzLog.

API Push Integration

NTA SkyNav sendet Flugdaten direkt an einen CapzLog-API-Endpunkt nach jeder gespeicherten Fahrt.

  • Benutzer verknüpft seinen CapzLog-Account einmalig (API-Key oder OAuth)
  • „An CapzLog senden“-Button erscheint nach dem Speichern einer Fahrt
  • NTA mappt die Flugdaten auf das erwartete CapzLog-Format
  • Echtzeit-Sync, kein manueller Export nötig

Benötigt von CapzLog: API-Dokumentation, Authentifizierungsmethode, erwartetes Datenformat

Nächste Schritte

Um mit der Integration fortzufahren, benötigen wir folgende Informationen von CapzLog:

  • API-Dokumentation — Bietet CapzLog eine REST-API für den Flugdaten-Import?
  • Datenformat-Spezifikation — Welche Felder und welches Format erwartet CapzLog?
  • Authentifizierungsmethode — API-Key, OAuth oder anderes?
