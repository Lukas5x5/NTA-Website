Some of you reported altitude issues — we already identified the cause and a fix is going live within the next few hours.

 

What's fixed in v1.0.1

Fix — Altitude Display

Altitude now uses GPS + EGM96 exclusively instead of the barometric sensor. The iOS barometer was returning incorrect absolute values, causing inaccurate altitude readings.

Fix — Android Variometer

Added barometer plugin for the variometer on Android devices (TYPE_PRESSURE sensor). Climb and descent rate should now work correctly on supported devices.

 

When will I get the update?

The update is being rolled out within the next few hours. Android users will receive it via Google Play, iOS users via TestFlight. Keep auto-updates enabled and you'll get it automatically.
